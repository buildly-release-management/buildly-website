GitHub Pages Setup

Easy step-by-step guide to deploy your CMS-powered website on GitHub Pages

1

Create GitHub Repository

Create a new repository on GitHub for your website:

Repository naming options:

  • ‚Ä¢ username.github.io - For your main personal/org site

3

Enable GitHub Pages

Configure GitHub Pages in your repository settings:

  1. 1. Go to your repository on GitHub
  2. 2. Click "Settings" tab
  3. 3. Scroll down to "Pages" section
  4. 4. Under "Source", select "Deploy from a branch"
  5. 5. Choose "main" branch and "/ (root)" folder
  6. 6. Click "Save"

üí° Tip: It may take a few minutes for your site to become available after enabling GitHub Pages.

4

Configure CMS for GitHub Pages

Update your admin page configuration:

Add to your admin.html:

// Auto-detected configuration for GitHub Pages
BuildlyCMS.init({
    site: {
        name: "My GitHub Pages Site",
        url: "https://username.github.io/repo-name"
    },
    system: {
        basePath: "/repo-name/buildly-cms/",
        githubPages: true
    }
});

üîß Troubleshooting

Common Issues:

  • 404 Error: Check that buildly-cms folder is in the root of your repository
  • Path Issues: Ensure all links use relative paths (no leading /)
  • CORS Errors: GitHub Pages automatically handles CORS for same-origin requests
  • Changes Not Showing: It may take 5-10 minutes for changes to deploy

Testing Locally:

# Serve locally to test
python -m http.server 8080
# or
npx serve .
# or
php -S localhost:8080

Custom Domain (Optional):

To use a custom domain like yoursite.com:

  1. 1. Add a CNAME file to your repo root with your domain
  2. 2. Configure DNS to point to username.github.io
  3. 3. Enable HTTPS in GitHub Pages settings
